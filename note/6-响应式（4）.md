### 清理 link 节点

#### 记录不能复用的节点

解决方案：

首先在创建 link3 的时候，把 link3 的 nextDep 指向 link2，当 depsTail 指向的 link 节点还有 nextDep 的时候，就把他清理掉

```ts
/**
 * 链接ref和effect，建立ref和effect之间的链表关系
 * @param dep ref对象
 * @param sub effect函数（依赖项）
 */
export function link(dep, sub) {

  const currentDep = sub.depsTail
  const nextDep = currentDep === undefined ? sub.deps : currentDep.nextDep
  if (nextDep && nextDep.dep === dep) {
    sub.depsTail = nextDep
    return
  }

  // 如果activeSub有，那就保存，等更新的时候触发
  const newLink: Link = {
    dep,
    sub,
    nextSub: undefined,
    prevSub: undefined,
    nextDep, // 这里的nextDep是复用失败的,在这个案例里面，复用name失败了
  }

  ...

}

...

```

上面的 newlink 中 nextDep 是复用失败的，结合前面的案例，其实复用失败的是 name 的节点，因为当点击 flag 后代码执行这里后

```ts
if (flag.value) {
  app.innerHTML = name.value
} else {
  app.innerHTML = age.value
}
```

这里会执行 else，也就是会走到 age 的访问器，当 age 走到 link 中的时候，sub.depsTail 实际上是 flag 节点，flag 节点的下一个节点就是 name 节点，然后

```ts
  if (nextDep && nextDep.dep === dep)
```

这个判断显然是不成立的，因为这里的 dep 是 age，而 nextDep 是 name 节点，所以复用失败，也就指的是这里了，因为复用失败了，所以这里需要创新创建 age 的节点，并且按照前面的思路，把 link 节点的 nextDep 指向当前复用失败的节点，也就是 name 节点。这也就是上面那段代码的意义了。
这样操作完以后，nextDep 就是需要清理的节点，那么接下来清理掉

#### 清理不能复用的节点，并且重新整理链表

```ts
function endTrack(sub) {
  const depsTail = sub.depsTail

  if (depsTail) {
    /**
     * depsTail有，并且depsTail还有nextDep，清理掉依赖关系
     */
    if (depsTail.nextDep) {
      clearTracking(depsTail.nextDep)
      depsTail.nextDep = undefined
    }
  } else if (sub.deps) {
	// 进入这里的场景是，一个依赖都没有了，也没有复用，也没有创建link，那么depsTail就不会有值，那么就意味着之前收集的所有的依赖都没有用上，那就可以清理掉了
    /**
     * 进入这里的场景是：
     * let e = effect(() => {
     *  if (count > 0) return
     *  count++
     *  if (flag.value) {
     *    app.innerHTML = name.value
     *  } else {
     *    app.innerHTML = age.value
     *  }
     * })
     * 在第一次的时候effect里面会收集到依赖，但是count发生变化的时候，这里的依赖都应该删掉
     */
    clearTracking(sub.deps)
    sub.deps = undefined
  }
}
```

这里新增了一个函数，并且有一个判断，如果 sub 的尾节点还有的话，那么把尾节点后面的清理掉，如果尾节点不存在，那么看一下有没有头节点，如果有头节点，那么全部清理掉。为什么呢？effect 执行完了以后 depsTail 一定会有值的，如果没有值，就只能意味着 effect 中没有收集到依赖，所以整个 sub 就都没有节点，所以需要把所有的 link 节点清理掉。

```ts
/**
 * 清理依赖
 * @param link
 */
function clearTracking(link: Link) {
  while (link) {
    const { dep, nextSub, nextDep, prevSub } = link
    /**
     * 如果上一个节点有，那就把上一个prevSub的下一个节点指向当前节点的下一个
     * 如果没有，那就是头节点，把subs指向下一个就行
     */
    if (prevSub) {
      prevSub.nextSub = nextSub
      link.nextSub = undefined
    } else {
      dep.subs = nextSub
    }

    /**
     * 如果下一个有，那就把next的上一个节点，指向当前节点的上一个节点
     * 如果下一个节点没有，他就是尾节点，那把dep的depsTail指向上一个
     */
    if (nextSub) {
      nextSub.prevSub = prevSub
      link.prevSub = undefined
    } else {
      dep.subsTail = prevSub
    }
    link.dep = link.sub = undefined
    link.nextDep = undefined
    link = nextDep
  }
}

```

#### 怎么清理？

解释一下清理的过程：

```ts
if (prevSub) {
  prevSub.nextSub = nextSub
  link.prevSub = undefined
} else {
  dep.subs = nextDep
}

if (nextSub) {
  nextSub.prevSub = prevSub
  link.prevSub = undefined
} else {
  dep.subsTail = prevSub
}
```

如果有上一个节点，那么把上一个节点的下一个节点改成下一个节点，再把当前节点的下一个节点改成 undefined，看一下图 [[6-响应式（4）-C2379B469533125DEF110399F94D4EAD.excalidraw]]

如果当前是头节点的话： [[6-响应式（4）-8805B1B9A8CD126F9768550F20093CE0.excalidraw]]
如果当前是尾节点的话，直接把 dep 的尾节点设置为 undefined

最后还需要把 link 节点进行重新赋

```ts
link.dep = link.sub = undefined
link.nextDep = undefined
link = nextDep
```

```ts
/**
 * 传播更新
 * @param subs 依赖项
 */
export function propagate(subs) {
  let link = subs
  const queueEffect = []
  while (link) {
    const sub = link.sub

    // 如果当前的订阅者没有被追踪，就跳过
    // 因为如果当前的订阅者没有被追踪，说明当前的订阅者没有被使用，所以不需要触发更新
    // 如果当前的订阅者被追踪，说明当前的订阅者被使用了，所以需要触发更新
    if (!sub.tracking) {
      queueEffect.push(sub)
    }
    link = link.nextSub
  }
  queueEffect.forEach(effect => effect.notify())
}
```
