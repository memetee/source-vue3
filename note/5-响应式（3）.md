其实简单来说，每一次执行 effect 中的 sub 的时候就会触发一次 getter，getter 会创建一个 link 节点，这个 link 节点就会保存当前的 sub
，然后这个 link 节点就会加入到 ref 的 subsTail 中，点击三次之后 flag 的链表一共有：初始化一次、点击第一次后共计 2 次，点击第二次后共计四次，点击第三次后共计八次。也就是说点击第四次的时候，会打印八次。
![[5-响应式（3）-0F7983F2D21732B06BAC21F30EFC1DF1.png]]
可以看一下打印，这里共有 8 个 sub 了
这就有问题了，重复收集并触发
解决方案也很简单：第一次收集的时候，知道收集的是什么依赖，那么下一次重新执行的时候，就不收集了，尝试复用之前的链表节点

现在的情况是 ref 的 subs 是一个链表，subs 是链表的头节点，然后节点有一个 sub，指向的是 effect，在源码里面是这样做的，effect 他也有一个链表，链表的头节点是 deps，deps 是一个头节点指向的是 link 节点，也就是 ref 的 sub 节点，并且给 link 节点新增一个 dep，指向的是 ref，画图出来是这样：[[5-响应式（3）-AF64E2E22326F9A75C392D368C630EDC.excalidraw]]

packages\reactivity\src\system.ts

```ts
/**
 * 链表节点
 */
export interface Link {
  // 保存effect
  sub: ReactiveEffect

  // 下一个节点,如果没有就是undefined
  nextSub: Link | undefined

  // 上一个节点,如果没有就是undefined
  prevSub: Link | undefined

  // 订阅者
  dep: Dep

  // 下一个订阅节点
  nextDep: Link | undefined
}

/**
 * 依赖项
 */
interface Dep {
  // 发布者链表的头节点
  subs: Link | undefined

  // 发布者链表的尾节点
  subsTail: Link | undefined
}

/**
 * 链接ref和effect，建立ref和effect之间的链表关系
 * @param dep ref对象
 * @param sub effect函数（依赖项）
 */
export function link(dep, sub) {
  const newLink: Link = {
    sub,
    dep,
    nextSub: undefined,
    prevSub: undefined,
    nextDep: undefined,
  }

	...

  /**
   * 将链表节点和sub建立关联关系(这里是单向链表)
   * 把新的依赖项添加到链表的尾部
   1. 如果链表不为空，就把新的依赖项添加到链表的尾部
   2. 如果链表为空，就把新的依赖项作为链表的头节点和尾节点
   */
  if (sub.depsTail) {
    sub.depsTail.nextDep = newLink
    sub.depsTail = newLink
  } else {
    sub.deps = newLink
    sub.depsTail = newLink
  }
}
```

这里给每一个 link 增加一个 dep，之前我们 link 保存了 sub，然后每次 ref 发生了更新，通过 link 中的 sub 找到 effect 的依赖，并且执行，现在不仅要在 link 中保存 sub，还要在 link 中保存 dep，也就是说，通过 effect 的依赖中的头节点能找到每一个依赖这个 effect 的依赖的 dep（也就是 ref）

> 注意：发布者是 ref，也就是 dep，订阅者是 sub，也就是副作用函数

## 复用 link 节点

packages\reactivity\src\effect.ts

```ts
export class ReactiveEffect {
    ...
  run() {
        ...
    /**
     *  这里设置为undefined是做一个标记。如果头节点为undefined，
     * 尾节点也是undefined表示，可以复用link，
     * 但凡头节点不是undefined表示可以复用link了
     */
    this.depsTail = undefined
    try {
      return this.fn()
    } finally {
      activeSub = prevSub // 这里恢复之前的effect函数，这样在处理嵌套的逻辑的时候就能正确地收集依赖了
    }
  }
	...
}
```

在执行副作用函数之前，先做一个标记，表示如果一个 sub 的 deps 有值，但是他的 depsTail 没有值，表示这个 link 是可以被复用的。

有了这个约定之后，我们就可以去创建 link 的时候进行判定处理了
packages\reactivity\src\system.ts

```ts
	...

/**
 * 链接ref和effect，建立ref和effect之间的链表关系
 * @param dep ref对象
 * @param sub effect函数（依赖项）
 */
export function link(dep, sub) {
  const currentDep = sub.depsTail
  const nextDep = currentDep === undefined ? sub.deps : undefined
  // 如果当前的订阅者的头节点有值，尾节点为undefined，说明可以复用link
  if (nextDep && nextDep.dep === dep) {
    sub.depsTail = nextDep
    return
  }
    ...
}

```

此时的链表结构大概是这样的 [[5-响应式（3）-F33E739614C4E94081D23C9817A0E905.excalidraw|5-effect的尾节点是undefined]]
代码的执行流程大概是这样的 [[5-响应式（3）-31F43256F9E3E2CBFABD52AC96607875.excalidraw]]

到这里，就可以解决这种多次收集 link，然后触发多个 link 的问题。这里通过复用 link 的方式实现，复用利用到了 sub 的 deps 链表和 dep 的 subs 链表，只要 subs 的 dep 链表节点中存在当前触发的 dep，那么表示已经收集过了这个 dep 依赖，不必重新收集依赖

      }
    </script>

点击了三次
![[5-响应式（3）-1B68280280583442984C876C1A04443A.png]]

可以发现点击了三次，触发了八次，原因是什么呢？我们能够阻止 flag 的重复创建 link，但是没有阻止 count 的重复创建。
其实最重要的就是每一次进入到 effect 函数中的时候，执行 sub 订阅者的时候，会先把 depsTail 设置为 undefined，然后执行完 flag 之后 depsTail 会被设置为 nextDep，然后 count 进来的时候，depsTail 已经不是 undefined 了，它就不会复用，而是重新创建 link
跟着代码详细看一下就是这样:[[5-响应式（3）-94B72B458103B489DEAE14D055997556.excalidraw]]

接下来就是解决 count 的复用问题了，先来看下在 flag 执行完以后，此时 effect 链表的 depsTail 指向了 link1
结构是这样的：[[5-响应式（3）-1DD0715A228C1B1AC558F7177EF223FD.excalidraw]]
因为 effect 的尾节点是有值的，而不是 undefined，所以不能复用 count 的 link 节点了
解决方案是：如果 sub 的尾节点 depsTail 的 nextDep 有值的话，那么使用 nextDep 进行复用，并且把尾节点指向这个 nextDep
结构是这样的：[[尾节点是depsTail的nextDep]]

packages\reactivity\src\system.ts

```ts
	...

/**
 * 链接ref和effect，建立ref和effect之间的链表关系
 * @param dep ref对象
 * @param sub effect函数（依赖项）
 */
export function link(dep, sub) {
  // region 尝试复用链表节点
  const currentDep = sub.depsTail

  // 分两种情况，如果头节点有，尾节点没有，那么尝试复用头节点
  // 如果尾节点还有nextDep,尝试复用尾节点的nextDep
  const nextDep = currentDep === undefined ? sub.deps : currentDep.nextDep
  // 如果当前的订阅者的头节点有值，尾节点为undefined，说明可以复用link
  if (nextDep && nextDep.dep === dep) {
    sub.depsTail = nextDep
    return
  }
    ...
}

	...

```

把判断改成这样的话，就能在 count 的时候也能复用了
