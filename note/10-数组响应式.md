## 长度发生修改

当前在监听数组的变化用 reactive 是可以正常监听到
例如某一个索引发生变化的话，可以监听到。但是如果修改了索引，就监听不到了

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <style>
      body {
        padding: 150px;
      }
    </style>
  </head>
  <body>
    <div id="app"></div>
    <script type="module">
      // 我们自己的
      import {
        reactive,
        ref,
        effect,
        computed,
        watch,
      } from '../dist/reactivity.esm.js'

      // 官方的
      // import {
      //   reactive,
      //   effect,
      //   ref,
      //   computed,
      //   watch,
      // } from '../../../node_modules/vue/dist/vue.esm-browser.prod.js'

      const state = reactive(['a', 'b', 'c', 'd'])

      /**
       * 更新数组的 length
       * 更新前 length = 4 => ['a', 'b', 'c', 'd']
       * 更新后 length = 2 => ['a', 'b']
       * 得出结论，要通知 访问了 c 和 d 的 effect 重新执行，就是大于等于length的索引重新执行
       */
      effect(() => {
        console.log('state[3] =>', state[3])
      })

      setTimeout(() => {
        // state[3] = 'e'
        state.length = 2 // ['a', 'b']
        console.log('执行setTimeout')
      }, 1000)
    </script>
  </body>
</html>
```

![[10-数组响应式-3568AF31EF8A3C1EC656C85D6F5622B9.png]]
这里更新了 length 的长度，造成数组发生了修改，但是会发现，并没有重新执行 effect，此时的 state[3] 是 undefined 了
正常来讲，修改了索引的话，要通知收集了大于等于索引的 effect 重新执行

packages\reactivity\src\dep.ts

```ts
export function trigger(target, key) {
  const depsMap = targetMap.get(target)

  if (!depsMap) {
    return
  }

  const targetIsArray = Array.isArray(target)
  if (targetIsArray && key === 'length') {
    /**
     * 更新数组的 length
     * 更新前 length = 4 => ['a', 'b', 'c', 'd']
     * 更新后 length = 2 => ['a', 'b']
     * 得出结论，要通知 访问了 c 和 d 的 effect 重新执行，就是大于等于length的索引重新执行
     * depsMap = {
     *  0：Dep,
     *  1: Dep,
     *  2: Dep,
     *  3: Dep,
     *  length: Dep
     * }
     */
    const length = target.length
    depsMap.forEach((dep, depKey) => {
      if (depKey >= length) {
        // 通知访问了大于等于length的重新执行
        propagate(dep.subs)
      }
    })
  } else {
    // 不是数组，或者数组更新的不是length
    /**
     * 找到 key 对应的Dep
     * key => Dep
     */
    const dep = depsMap.get(key)
    if (!dep) {
      // dep 不存在，表示这个key没有在sub中访问过
      return
    }

    /**
     * 找到 dep 的 subs 通知他们重新执行
     */
    propagate(dep.subs)
  }
}
```

这里代码判断了是否修改 length，如果修改了 length，就去通知所有大于 length 的地方重新执行。按照这样修改的话，就能在修改 length 为 2 之后，就通知访问了 state[3]、state[4] 的地方重新执行

## 监听 push

length 也有可能通过 push 被动更新

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body>
    <div id="app"></div>
    <script type="module">
      import {
        reactive,
        ref,
        effect,
        computed,
        watch,
      } from '../dist/reactivity.esm.js'

      const state = reactive(['a', 'b', 'c', 'd'])

      /**
       * 隐式更新length
       * 更新前: length = 4 => ['a', 'b', 'c', 'd'];
       * 更新前: length = 5 => ['a', 'b', 'c', 'd', 'e'];
       * 更新动作，以 push 为例，追加了一个 e
       * 隐式更新 length 的方法： push pop shift unshift
       * 
       */
      effect(() => {
        console.log('state[3] =>', state.length)
      })

      setTimeout(() => {
        state.push('e')
      }, 1000)
    </script>
  </body>
</html>
```

packages\reactivity\src\baseHandlers.ts

```ts
export const mutableHandlers = {
    ...

  set(target, key, newVal, receiver) {
    const oldVal = target[key] // 旧的值（还没更新）

    // 判断是否为数组
    const targetIsArray = Array.isArray(target)

    // 拿到旧的length
    const oldLength = targetIsArray ? target.length : 0

    // 处理ref的赋值场景：如果旧值是ref，新值不是ref，直接给ref的value赋值
    if (isRef(oldVal) && !isRef(newVal)) {
      oldVal.value = newVal
      return true
    }

    // 更新代理对象的值：先完成值的更新，再触发依赖，确保依赖能获取到最新值
    const success = Reflect.set(target, key, newVal, receiver)

    // 判断是否有修改，有修改的话就传播更新
    if (hasChanged(newVal, oldVal)) {
      trigger(target, key)
    }

    // 获取更新后的length
    const newLength = targetIsArray ? target.length : 0

    // 只有数组，并且length有修改，并且，修改的不是length属性
    if (targetIsArray && newLength !== oldLength && key !== 'length') {
      /**
       * 隐式更新length
       * 更新前: length = 4 => ['a', 'b', 'c', 'd'];
       * 更新后: length = 5 => ['a', 'b', 'c', 'd', 'e'];
       * 更新动作，以 push 为例，追加了一个 e
       * 隐式更新 length 的方法： push pop shift unshift
       */
      trigger(target, 'length')
    }
    return success
  },
}
```

这里通过判断是否 length 被动修改，这里需要传播更新用到 length 的地方
