## 创建项目

### 初始化项目

使用 pnpm+monorepo 管理项目依赖，先初始化一个项目。

```bash
pnpm init
```

### 创建 monorepo 项目

想要使用 monorepo 管理项目需要创建一个文件叫 `pnpm-workspace.yaml`，并且要告诉 monorepo 管理的目录是哪一个。
(新增)code\pnpm-workspace.yaml

```yaml
packages:

  - "packages/*"
```

### 当前目录

创建 packages 目录，创建完以后，当前项目目录就是这样了

```
├─📁 packages/-------------- # 包
├─📄 package.json----------- # 项目包
└─📄 pnpm-workspace.yaml---- # monorepo管理配置
```

### 下载 TS

配置完 monorepo 之后，下载一个依赖 (之前可能需要再后面加 -w 但是现在不需要了)

```bash
pnpm install typescript @types/node -D
```

### 初始化 TS、配置 TS

执行 ts 初始化（执行当前项目初始化，而不是全局，所以这里用的是 npx）

```bash
npx tsc --init
```

配置 ts
code\tsconfig.json

```json
{
  "compilerOptions": {
    "target": "ESNext", // 指定 ECMAScript 目标版本
    "module": "ESNext", // 指定模块代码生成规范
    "moduleResolution": "node", // 指定模块解析策略
    "outDir": "dist", // 指定编译输出的目录
    "resolveJsonModule": true, // 允许导入 JSON 文件
    "strict": false, // 关闭严格模式
    "lib": ["ESNext", "DOM"], // 指定要使用的库文件
    "baseUrl": "./"
  }
}
```

## 配置 dev 开发环境打包

如何配置？配置开发环境需要考虑哪些？
首先要知道，在进行打包的时候，需要打包出来什么样的文件，例如 esm，cjs 等，所以需要我们在执行打包的时候能提供不同的参数，打包出来不同的文件，例如：`node scripts/dev.js --format esm` 生成的是 esm 类型的文件，`node scripts/dev.js --format cjs` 生成的是 cjs 类型的文件

### 区分生成文件类型

code\package.json

```json
	...

  "scripts": {
    "dev": "node scripts/dev.js --format cjs"
  }

	...
```

想要拿到这些传过来的参数，可以用 `process.arg`，拿到的格式是这样的

```bash
$ pnpm run dev

> Vue3@1.0.0 dev C:\wuwuwu\Vue\Vue3
> node scripts/dev.js --format cjs

[
  'C:\\nvm4w\\nodejs\\node.exe',
  'C:\\wuwuwu\\Vue\\Vue3\\scripts\\dev.js',
  '--format',
  'cjs'
]
```

可以看到拿到了 `--format` 和 `cjs`，在数组里面。
但是我们想要的是对象，那么可以通过这种方式来获取

code\scripts\dev.js

```js
/**
 * 打包开发环境
 * 
 * node scripts/dev.js --format cjs
 */
import { parseArgs } from 'node:util'
const { values } = parseArgs({
  options: {
    format: {
      type: 'string', // 选项的类型
      short: 'f', // 选项的短名称
      default: 'esm', // 选项的默认值
    },
  },
})
console.log(values)
```

这样就可以拿到传递过来的参数，打印出来的结果是

```bash
$ pnpm run dev

> Vue3@1.0.0 dev C:\wuwuwu\Vue\Vue3
> node scripts/dev.js --format cjs

[Object: null prototype] { format: 'cjs' }
```

可以看到 value 就是一个对象，在 format 后面接的 cjs 就能拿到

### 区分打什么包（位置参数）

上面做完之后，我们就能正确解析出传过来的参数了，但是还有一个问题，就是 monorepo 管理的项目可能有很多包，比如 reactivity 包（创建一个文件夹叫 reactivity），vue 包（创建一个文件夹叫 vue），打哪个包呢？所以还需要一个位置参数

code\package.json

```json
	...

"scripts": {
	"dev": "node scripts/dev.js reactivity --format cjs"
},

	...
```

这里传了 reactivity，接着去拿这个参数
code\scripts\dev.js

```js
import { parseArgs } from 'node:util'

const {
  values: { format },
  positionals,
} = parseArgs({
  allowPositionals: true, // 允许位置参数

	...

})
console.log(format, positionals)
```

打印结果

```bash
$ npm run dev

> Vue3@1.0.0 dev
> node scripts/dev.js reactivity --format cjs

cjs [ 'reactivity' ]
```

这里加了 `allowPositionals` 参数，就能拿到我们传递的 `reactivity`

接下来要拿到入口文件，上面写的 reactivity 实际上想做的是要打包 reactivity 这个包的文件，所以我们先要拿到目标的打包文件，对这个目标打包文件进行打包
code\scripts\dev.js

```js
	...

const {
  values: { format },
  positionals,
} = parseArgs({

	...

})

// 如果有获取到要打包的目标，否则默认打包整个 vue
const target = positionals.length ? positionals[0] : 'vue'
```

这样就可以拿到我们要打包的名字了，接着创建这些包。

### 配置 esbuild 打包

知道要打什么包，并且拿到该包的路径，就可以进行 esbuild 配置了

```js
import esbuild from 'esbuild'

...

esbuild
  .context({
    entryPoints: [entry], // 入口文件
    outfile: resolve(
      __dirname,
      `../packages/${target}/dist/${target}.${format}.js`, // 针对不同的格式输出不同的文件
    ), // 输出文件
    format, // 输出格式
    platform: format === 'cjs' ? 'node' : 'browser', // 目标平台
    sourcemap: true, // 生成 source map
    bundle: true, // 打包所有依赖到一个依赖中
  })
  .then(ctx => {
    ctx.watch() // 监听文件变化，自动重新打包
  })
```

### IIFE 打包

如果打包的是 IIFE，这种 IIFE 需要对外提供一个全局变量，所以针对这个也需要处理
需要在 reactivity 文件夹下创建一个文件叫 `package.json` 的文件，并且在里面写配置，这里的配置参考的是 vue 的配置
在 node_modules 中找到 reactivity，然后拿到它的 package.json，进行参考
这里给出删减过后我们需要的
code\packages\reactivity\package.json

```json
{
  "name": "@vue/reactivity",
  "version": "1.0.0",
  "description": "响应式模块",
  "main": "dist/reactivity.cjs.js",
  "module": "dist/reactivity.esm.js",
  "files": ["index.js", "dist"],
  "sideEffects": false,
  "buildOptions": {
    "name": "VueReactivity",
    "formats": ["esm-bundler", "esm-browser", "cjs", "global"]
  }
}
```

> 注意：这里有一个配置叫 `buildOptions`，它不是 package 的配置，是 vue 自己往里添加的，他的作用就是在使用 cdn 引入的时候，会往 window 里面加一些属性。
> 例如：使用 window.vue.createApp，那么 name 就是 vue，如果 name 是 VueReactivity，那么就是 window.VueReactivity。
> 这里的 formats 是生产环境用的，生产环境打出来的包就是这些了

导入 reactivity 的 package.json，注意，因为 require 它是 commonJS 的，在 ESModule 中没有，和上面的 `__dirname` 一样，所以我们也需要重新获取 require
code\scripts\dev.js

```js

	...

const __dirname = dirname(__filename);

// 配置入口文件

const pkg = require(`../packages/${target}/package.json`)

	...
```

### monorepo 使用依赖包

monorepo 管理的项目获取依赖包和我们平常使用依赖包不同，例如：我们再 shared/src/index.ts 中写一个工具函数
code\packages\shared\src\index.ts

```ts
export function isObject(value: any): value is Record<any, any> {
  return value !== null && typeof value === 'object'
}
```

在这里 export 导出了一个函数叫 isObject，我们希望在 reactivity 中使用
想在 reactivity 中使用，需要先下载这个依赖包

```bash
pnpm install @vue/shared --workspace --filter @vue/reactivity
```

这里的**workpace**表示使用 monorepo 中的包，**--filter**表示安装到@vue/reactivity 这个项目中

安装完了以后就可以在 reactivity 项目的 package.json 中看到
code\packages\reactivity\package.json

```json
{
	...

  "dependencies": {
    "@vue/shared": "workspace:*"
  }
}
```

这样就安装成功了，接着测试一下
code\packages\reactivity\package.json

```json
import { isObject } from '@vue/shared'

isObject({})
```

但是这样写会报错
![[1-搭建环境-报错.png]]

这里是 ts 找不到，所以需要配置一下
code\tsconfig.json

	  ...

## 开发环境配置完整代码

code\scripts\dev.js

```js
/**
 * 打包开发环境
 * 
 * node scripts/dev.js --format cjs
 */
import { dirname, resolve } from 'node:path'
import { fileURLToPath } from 'node:url'
import { parseArgs } from 'node:util'
import esbuild from 'esbuild'
import { createRequire } from 'node:module'

/**
 * 解析命令行参数
 */
const {
  values: { format },
  positionals,
} = parseArgs({
  allowPositionals: true, // 允许位置参数
  options: {
    format: {
      type: 'string', // 选项的类型
      short: 'f', // 选项的短名称
      default: 'esm', // 选项的默认值
    },
  },
})

// 创建esm的__filename和__dirname
const __filename = fileURLToPath(import.meta.url)
const require = createRequire(import.meta.url)
const __dirname = dirname(__filename)

// 如果有获取到要打包的目标，否则默认打包整个 vue
const target = positionals.length ? positionals[0] : 'vue'

// 配置入口文件
const entry = resolve(__dirname, `../packages/${target}/src/index.ts`)

// 导入package.json
const pkg = require(`../packages/${target}/package.json`)

esbuild
  .context({
    entryPoints: [entry], // 入口文件
    outfile: resolve(
      __dirname,
      `../packages/${target}/dist/${target}.${format}.js`, // 针对不同的格式输出不同的文件
    ), // 输出文件
    format, // 输出格式
    platform: format === 'cjs' ? 'node' : 'browser', // 目标平台
    sourcemap: true, // 生成 source map
    bundle: true, // 打包所有依赖到一个依赖中
    globalName: pkg.buildOptions?.name, // 全局变量名称，umd和iife格式需要
  })
  .then(ctx => {
    ctx.watch() // 监听文件变化，自动重新打包
  })
```
