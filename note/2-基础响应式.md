## 什么是响应式

Vue 的响应式系统核心在于响应式对象的属性与 `effect` 副作用函数之间建立的依赖关系。

示例 1：
普通函数访问响应式数据

```js
import { ref } from 'vue'

const count = ref(0)

// 普通函数
function fn() {
  console.log(count.value)
}

fn() // 打印 0

setTimeout(() => {
  count.value = 1 // 修改值不会触发 fn 重新执行
}, 1000)
```

在这个例子中，虽然 `fn` 读取了响应式数据 `count.value`，但由于它不是在 effect 中执行的，因此当 `count.value` 发生变化时，该函数不会重新执行。

示例二：
`effect` 中访问响应式数据

```js
import { ref, effect } from 'vue'

const count = ref(0)

effect(() => {
  console.log(count.value) // 首次执行打印 0
})

setTimeout(() => {
  count.value = 1 // 触发 effect 重新执行，打印 1
}, 1000)
```

我们平时使用的 `computed`、`watch`、`watchEffect` 包括组件的 `render` 都是依赖于 `effect` 函数来收集依赖的
当在 effect 中访问响应式数据时，会发生以下过程：

- **依赖收集**：当 `effect` 中的函数首次执行时，访问 `count.value` 会触发 `ref` 的 `get`，此时系统会自动收集当前 `effect` 作为依赖。
- **触发更新**：当 `count.value` 被修改时，会触发 `ref` 的 `set`，系统会通知之前收集的所有依赖（`effect`）重新执行。

在第二个例子中，修改 `count.value` 会导致 effect 重新执行并打印新值。这种自动追踪依赖和触发更新的机制，正是 Vue 响应式系统的核心特征。

class RefImpl {

## IS_REF 属性

/packages\reactivity\src\ref.ts

```ts
export enum ReactiveFlags {
  IS_REF = '__v_isRef',
}

// ref对象的构造函数
class RefImpl {
  _value; // ref获取到的值（实际值）

  [ReactiveFlags.IS_REF] = true // 标记，表示这个对象是一个ref对象

  constructor(value) {
    this._value = value
  }
}

export function ref(value) {
  return new RefImpl(value)
}

/**
 * 判断一个对象是不是一个ref对象
 * @param value 对象
 * @returns 返回布尔，true表示是一个ref对象，否则是其他
 */
export function isRef(value) {
  return !!(value && value[ReactiveFlags.IS_REF])
}
```

新增枚举，并且在 RefImpl 类中新增了 [ReactiveFlags.IS_REF] = true 表示它是一个 ref 对象，所有通过这个构造函数创建的对象都是 ref 对象

## 收集依赖

上述虽然可以监听到访问和修改，但是没有做到数据修改后，自动触发引用了该依赖项的依赖重新执行；

这里需要解决一些术语上的问题，什么是依赖？什么是依赖项？什么又是收集依赖？
依赖：可以理解为，一段需要执行的副作用代码，也就是 effect 中接收的需要执行的函数，它也可以被称为**发布者**
依赖项：可以理解为，被依赖的变量，例如上面的 count，它也可以被称为**订阅者**、**观察者**
收集依赖：收集需要执行的副作用的代码，在依赖项发生变化的时候，执行这段代码

packages\reactivity\src\effect.ts

```ts
// 全局变量，用来保存当前正在执行的依赖（订阅者），这里需要导出。
export let activeSub
export function effect(fn) {
  activeSub = fn // 保存依赖
  activeSub() // 执行依赖
  activeSub = undefined // 执行完依赖需要清空，避免污染
}
```

定义一个全局变量，用来保存当前正在触发的依赖，接着我们再 get 的时候是可以拿到这个依赖的值。

测试：

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body>
    <script type="module">
      import { ref, effect } from '../dist/reactivity.esm.js'
      const count = ref(0)
      effect(() => {
        console.log('count', count.value)
      })
      setTimeout(() => {
        count.value = 100
      }, 1000)
    </script>
  </body>
</html>
```

上面代码中，effect 传入的函数会保存到 activeSub 中，并且执行 activeSub，执行完以后清空...

在执行 activeSub 的时候，实际上会访问 count.value，也就是上面的打印，在访问的过程中，我们是可以在 get 中识别到的，所以

packages\reactivity\src\ref.ts

```ts
	...

class RefImpl {

	...

  get value() {
    console.log('访问了get', activeSub)
    return this._value;
  }

	...

}

	...
```

在 get 中加一个打印的话，是会触发的，当前拿到的 activeSub ，就是依赖（订阅者），也就是 effect 传入的函数。

现在能拿到依赖了，下一步就是保存这个依赖了

```ts

	...

class RefImpl {
  _value;

  [ReactiveFlags.IS_REF] = true

  subs // 用来保存和effect之间的关联关系，简单来讲就是保存依赖（订阅者）

  constructor(value) {
    this._value = value
  }

  get value() {
    if (activeSub) {
      // 如果activeSub有值的话，需要保存，等到更新的时候触发
      this.subs = activeSub
    }
    return this._value
  }

  set value(newVal) {
    this._value = newVal

    // 触发保存的依赖
    this.subs?.()
  }
}

	...

```

这里用 subs 来保存依赖，当访问 get 函数的时候，用 subs 进行保存，然后在 set 的时候进行触发更新

> 注意，在 set 的时候，这里的 subs 就是 activeSub，也就是依赖，也就是 effect 的回调函数。

以上就是响应式的基础代码了。
